工具让 LLM 能"动手"，不只是"动嘴"——解决实时信息、精确计算、外部系统访问的问题
Function Calling 是结构化输出，让 LLM 说"我需要调用什么"而不是瞎编答案
工具定义要清晰，尤其是 description，直接影响 LLM 的选择和填参准确率
生产环境必须考虑：超时、限流、安全（不要 eval）、成本追踪
工具数量要控制，太多会让 LLM 选择困难，按任务类型过滤到 3-5 个

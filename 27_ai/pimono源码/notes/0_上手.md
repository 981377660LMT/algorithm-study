## Pi Monorepo 项目全面讲解

帮我讲解整个项目，我应该如何学习、完全掌握。

---

### 一、项目定位

Pi 是一个**用于构建 AI Agent 的开源 TypeScript 工具集**，由 Mario Zechner（libGDX 作者）开发。它的核心产物是一个**终端交互式编程助手**（类似 Claude Code / Cursor），但设计上是分层的 monorepo，每一层都可以独立使用。

---

### 二、架构总览（自底向上的分层）

```
┌──────────────────────────────────────────────────────┐
│             用户交互层 (Frontends)                      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐            │
│  │coding-agent│ │  web-ui  │  │   mom    │            │
│  │(终端 CLI)  │ │(浏览器UI) │  │(Slack Bot)│            │
│  └─────┬─────┘ └────┬─────┘  └────┬─────┘            │
├────────┼────────────┼─────────────┼──────────────────┤
│        │            │             │                    │
│  ┌─────▼────────────▼─────────────▼─────┐             │
│  │         agent (Agent Runtime)         │             │
│  │   状态管理 / Tool 执行 / 事件流        │             │
│  └─────────────────┬─────────────────────┘             │
│                    │                                   │
│  ┌─────────────────▼─────────────────────┐             │
│  │          ai (统一 LLM API)             │             │
│  │ 多 Provider / 流式 / Tool Calling      │             │
│  └───────────────────────────────────────┘             │
├──────────────────────────────────────────────────────┤
│   辅助层                                               │
│   tui (终端UI框架)  │  pods (GPU Pod 部署)              │
└──────────────────────────────────────────────────────┘
```

**7 个包**，依赖关系从下到上：

| 层级          | 包名              | 作用                                                                                                   |
| ------------- | ----------------- | ------------------------------------------------------------------------------------------------------ |
| **L0 基础**   | `pi-ai`           | 统一的多 Provider LLM API（OpenAI/Anthropic/Google 等 20+ 个），处理流式输出、Tool Calling、Token 计费 |
| **L0 基础**   | `pi-tui`          | 终端 UI 框架，差量渲染、组件化（Editor/Markdown/SelectList 等）                                        |
| **L1 运行时** | `pi-agent-core`   | Agent 运行时：状态机 + agentLoop（prompt → LLM → tool execution → 循环），事件流                       |
| **L2 应用**   | `pi-coding-agent` | **核心产品**：交互式终端编程助手，内置 read/write/edit/bash 工具，支持 Extension/Skill/Theme           |
| **L2 应用**   | `pi-web-ui`       | 浏览器版 Chat UI 组件（Web Components + Tailwind）                                                     |
| **L2 应用**   | `pi-mom`          | Slack Bot，将消息委托给 pi agent，能自己装工具、写脚本                                                 |
| **独立**      | `pi-pods`         | GPU Pod 管理 CLI，部署 vLLM 到远程 GPU 服务器                                                          |

### 三、各包核心机制详解

#### 1. `pi-ai` — 统一 LLM API （最底层，最重要）

**关键文件：**

- types.ts — 核心类型定义：`Model`, `Context`, `Message`, `Tool`, `StreamOptions`, `Usage`
- api-registry.ts — Provider 注册中心（注册/查找 API 实现）
- models.ts — `getModel()` 获取模型，`calculateCost()` 计算成本
- stream.ts — `stream()` / `complete()` / `streamSimple()` 核心入口
- providers/ — 每个 Provider 的具体实现（anthropic.ts, google.ts, openai-responses.ts 等）

**核心设计模式：**

```
registerApiProvider(anthropicProvider)   // 启动时自动注册
getModel("anthropic", "claude-sonnet-4") // 获取 Model 对象
stream(model, context)                   // 统一的流式调用
```

- **策略模式**：每个 Provider 实现相同接口（`stream`/`streamSimple`），通过 registry 注册
- **TypeBox Schema**：Tool 的参数用 TypeBox 定义，既是类型又是运行时校验

#### 2. `pi-agent-core` — Agent 运行时

**关键文件：**

- types.ts — `AgentMessage`, `AgentTool`, `AgentLoopConfig`, `AgentEvent`
- agent-loop.ts — **核心循环**：prompt → LLM → tool calls → tool results → 再次 LLM
- agent.ts — `Agent` 类，封装状态 + 循环 + 事件订阅

**核心循环 `agentLoop`：**

```
用户 prompt → [transformContext] → [convertToLlm] → LLM 调用
                                                      ↓
                                              assistant 回复
                                                      ↓
                                           有 tool calls？
                                          ↙            ↘
                                         否              是 → 执行 tools → tool results → 下一轮
                                         ↓
                                    检查 steering/followUp
                                         ↓
                                      agent_end
```

**关键抽象：**

- `AgentMessage` 比 LLM `Message` 更灵活，可以扩展自定义消息类型
- `convertToLlm` 在发送前过滤/转换为 LLM 能理解的格式
- `transformContext` 做上下文裁剪（防止 token 超限）
- **事件流**（EventStream）让 UI 可以实时响应

#### 3. `pi-coding-agent` — 交互式编程助手（核心产品）

**关键目录：**

- core/tools/ — 内置工具：`bash.ts`, `read.ts`, `write.ts`, `edit.ts`, `grep.ts`, `find.ts`, `ls.ts`
- core/agent-session.ts — 会话管理
- core/extensions/ — 扩展系统
- core/system-prompt.ts — 系统提示词构造
- core/compaction/ — 上下文压缩
- core/sdk.ts — SDK 入口，`createAgentSession()`
- modes/interactive/ — 交互式 TUI 模式
- modes/print-mode.ts — 非交互模式
- modes/rpc/ — RPC 服务模式

**四大可定制机制：**

1. **Extensions** — TypeScript 插件，可以添加工具、命令、UI 组件、快捷键
2. **Skills** — Markdown 文件，作为专业领域知识注入 system prompt
3. **Prompt Templates** — 可复用的提示词模板
4. **Themes** — 终端主题定制

#### 4. `pi-tui` — 终端 UI 框架

三级渲染策略的差量渲染引擎；组件包括 Editor（支持自动补全、语法高亮）、Markdown（终端渲染）、SelectList、Image（Kitty/iTerm2 协议）等。

#### 5. `pi-web-ui` — 浏览器 Chat UI

基于 mini-lit Web Components + Tailwind CSS v4，提供 ChatPanel、Artifacts（沙箱执行 HTML）、附件处理、IndexedDB 存储等。

#### 6. `pi-mom` — Slack Bot

把 Slack 消息转发给 pi agent，在 Docker 沙箱中执行，能自己安装工具和创建 CLI 脚本。

#### 7. `pi-pods` — GPU 部署

管理远程 GPU Pod 上的 vLLM 部署，支持 DataCrunch、RunPod 等。

### 四、推荐学习路径

按照**自底向上**的顺序，每个阶段都有明确的学习目标：

#### 阶段 1：`pi-ai`（预计 2-3 天）

**目标**：理解如何用统一接口调用不同 LLM Provider

1. 读 packages/ai/README.md 的 Quick Start 和 Tools 部分
2. 精读核心类型：types.ts — 理解 `Model`, `Context`, `Message`, `Tool`, `StreamOptions`
3. 精读 Provider 注册机制：api-registry.ts
4. 精读流式输出：stream.ts + utils/event-stream.ts
5. 选一个 Provider 深入：推荐 providers/anthropic.ts（最典型）
6. 跑测试：`npx tsx ../../node_modules/vitest/dist/cli.js --run test/` 加深理解

**练手**：自己写一个脚本，用 `getModel` + `stream` 调一个 LLM，传一个 Tool

#### 阶段 2：`pi-agent-core`（预计 1-2 天）

**目标**：理解 Agent 循环如何运作

1. 读 packages/agent/README.md 的事件流图
2. 精读 types.ts — `AgentMessage`, `AgentTool`, `AgentEvent`
3. **重点精读** agent-loop.ts — 理解 `runLoop` 函数：
   - LLM 调用 → 解析 tool calls → 并发执行 tools → 收集结果 → 下一轮
   - steering messages 和 follow-up messages 的机制
4. 读 agent.ts — 看 `Agent` 类如何封装 agentLoop

**练手**：用 `Agent` 类写一个带自定义 Tool 的简单 agent

#### 阶段 3：`pi-coding-agent`（预计 3-5 天，核心）

**目标**：理解完整的编程助手是如何搭建的

1. 先读 packages/coding-agent/README.md 中 Extensions/Skills 部分
2. 读 core/tools/ — 理解 read/write/edit/bash 四个核心工具的实现
3. 读 core/system-prompt.ts — 看 system prompt 如何拼装
4. 读 core/agent-session.ts — 会话生命周期
5. 读 core/compaction/ — 上下文压缩策略
6. 读 core/extensions/ — 扩展系统加载和运行机制
7. 读 core/sdk.ts — `createAgentSession()` 把所有东西串起来
8. 读 modes/interactive/ — 看 TUI 模式如何驱动 agent

**练手**：

- 写一个 Extension，添加一个自定义命令
- 用 SDK 模式嵌入到自己的 Node.js 应用中

#### 阶段 4：`pi-tui`（按兴趣，1-2 天）

**目标**：理解终端差量渲染

读 packages/tui/README.md 的 Rendering 部分，理解三种渲染策略（full/differential/content-only），然后看 Editor 组件的实现。

#### 阶段 5：`pi-web-ui` / `pi-mom` / `pi-pods`（按需）

这三个是上层应用，根据你的兴趣选读。

### 五、学习技巧

| 技巧             | 说明                                                                  |
| ---------------- | --------------------------------------------------------------------- |
| **跑起来再读**   | 先 `npm install && npm run build`，然后 `./pi-test.sh` 实际体验       |
| **看测试学用法** | 每个包的 `test/` 目录是最好的使用示例                                 |
| **画数据流图**   | 对 `agentLoop` 中 `Message → LLM → ToolCall → ToolResult` 的流转画图  |
| **断点调试**     | 用 VS Code 在 agent-loop.ts 的 `runLoop` 函数设断点，观察每一轮的数据 |
| **先宽后深**     | 先理解各包的 index.ts 导出了什么，再深入具体实现                      |
| **对比其他项目** | 可以对比 Vercel AI SDK、LangChain 的设计，理解 pi 的设计取舍          |

### 六、核心设计理念

1. **最小核心 + 扩展机制**：coding-agent 只内置 4 个工具（read/write/edit/bash），所有额外功能通过 Extension/Skill 实现
2. **分层解耦**：`ai` 层不知道 agent，agent 层不知道 coding-agent，每层可独立使用
3. **流式优先**：所有 LLM 交互都是 EventStream，UI 可以实时响应
4. **TypeBox 做 Schema**：Tool 参数定义既是 TypeScript 类型，又是运行时 JSON Schema 验证
5. **No MCP**：不使用 Model Context Protocol，而是用 TypeScript Extension + 自定义 Tool 的方式，更直接可控

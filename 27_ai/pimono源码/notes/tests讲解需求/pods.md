# pi (项目名称)

> **原文**：Deploy and manage LLMs on GPU pods with automatic vLLM configuration for agentic workloads.
> **新手解析**：这是工具的**核心一句话介绍**。它是一个帮助你在**云端 GPU 服务器（Pods）**上轻松部署和管理**大语言模型（LLMs）**的工具。它能够自动为你配置 `vLLM`（目前业界最流行、速度极快的大模型推理引擎），而且特别适合运行那些能使用工具、执行复杂任务的“智能体（Agent）”模型。

---

## 1. Installation (安装)

> **原文**：`npm install -g @mariozechner/pi`
> **新手解析**：告诉你要怎么安装它。你需要在你的电脑上安装好 `Node.js`（一个运行 JavaScript 的环境），然后打开终端，输入这行 `npm` 命令将 `pi` 全局安装到你的电脑上。

---

## 2. What is pi? (pi 到底能干啥？)

> **原文列出 5 个特性：**
>
> - Sets up vLLM on fresh Ubuntu pods
> - Configures tool calling for agentic models...
> - Manages multiple models on the same pod with "smart" GPU allocation
> - Provides OpenAI-compatible API endpoints for each model
> - Includes an interactive agent with file system tools for testing
>
> **新手解析**：这里回答了它为什么有价值：

1. **自动配置**：买了一台全新的 Ubuntu 系统的云服务器后，它能全自动帮你装好环境，不需要你自己瞎折腾配置。
2. **工具调用配置**：有些厉害的模型（比如 Qwen, GLM 等）支持调用外部工具（计算器、浏览器等），系统会自动把这些高级功能帮你打通。
3. **多卡智能分配**：如果你的服务器有多张显卡，它会“聪明地”把不同的模型分配到不同的显卡上。
4. **兼容 OpenAI 接口**：这是杀手锏！它能把你跑的开源大模型，伪装成跟 OpenAI (ChatGPT) 完全一样的 API。这意味着以往任何能连 ChatGPT 的软件，换个网址就能连你的私有模型。
5. **自带聊天测试窗**：内置了一个自带文件系统控制能力的聊天机器人，方便你测试刚刚跑起来的模型聪不聪明。

---

## 3. Quick Start (快速开始)

> **新手解析**：这里为你演示了工具跑起来的四个步骤：

1. **设置密码和钥匙 (export...)**：你需要提供 `HF_TOKEN`（去 HuggingFace 网站申请的密钥，用来下载模型）和 `PI_API_KEY`（你自己随便写的一个密码，用来防止别人偷用你待会建好的 API）。
2. **连接初始化服务器 (`pi pods setup...`)**：通过这行命令，用 SSH 远程连接上你的云服务器，如果云上有挂载的大容量硬盘也会帮你配置好。
3. **启动模型 (`pi start algorithm-study.`)**：一键拉取并运行著名的 Qwen2.5 编程大模型。
4. **对话测试 (`pi agent...`)**：直接在自己电脑终端发送问题测试，或者甚至拿它当作 OpenAI 的平替接口来写代码。

## 4. Prerequisites (前置条件)

> **新手解析**：在用它之前你必须具备的东西：

- 电脑上有 Node.js（18版本以上）。
- 有 HuggingFace 的 Token（免费注册获取）。
- 有一台云端 GPU 服务器，要求：装了 Ubuntu 系统、有 root（最高管理员）权限、装了 NVIDIA 显卡驱动、有一块足够大用来存模型的硬盘。

---

## 5. Supported Providers (支持的云厂商)

> **新手解析**：推荐了买显卡服务器的地方：

- **DataCrunch**：最推荐。因为它的云盘（NFS）可以挂载给多台机器共享。大模型动辄几十个G，挂一个共享盘，下载一次可以到处用。对于经常做实验的人最友好。
- **RunPod**：也不错，支持持久化存储（关机了硬盘数据还在），但不能多台机器同时用。
- 也提到 Vast.ai、AWS 等只要符合上述“前置条件”任何服务器都能用。

## 6. Commands (命令大全)

这是你以后日常用得最多的部分，分为了三个大类：

### 6.1 Pod Management (物理机器的管理)

> **新手解析**：

- `pi pods setup`：用来让工具记住你的某台服务器密码/端口并给它装环境。还可以选装 vLLM 引擎的版本（`release` 稳定版，`nightly` 新功能版）。
- `pi pods`：显示你现在总共绑定了几台机器。
- `pi pods active`：当你有多台机器时，切换当前对哪台机器发号施令。
- `pi shell / ssh`：一键连接登入服务器，不用每次敲长长的 ssh 密码了。

### 6.2 Model Management (大模型的管理)

> **新手解析**：

- `pi start` 启动指令，能带很多高级参数：
  - `--memory`：占用显卡的比例。防止它一口吞掉全部显存，导致别的软件会卡死。
  - `--context`：上下文长度。越大，它能一次性阅读的文章/代码就越长。
  - `--gpus`：指定用几个显卡来跑这个模型。
- `pi stop` 关掉模型、`pi list` 看哪些模型在跑。
- `pi logs`：查看模型的运行日志，这是出了问题用来找错误原因的神器。

### 6.3 Agent & Chat Interface (智能体与终端聊天界面)

> **新手解析**：
> 使用 `pi agent` 直接能跟你后台跑的模型产生互动。加上 `-i` 参数就可以像用微信一样在终端里跟它进行持续的一问一答聊天。甚至提供了 `pi-agent` 这个独立命令去当你的通用日常对话机器人。

## 7. Predefined Model Configurations (预设模型配置)

> **新手解析**：跑大语言模型通常需要一堆复杂的参数。`pi` 为了照顾新手，给业界目前最好用、带有“智能体性质”的模型写好了**预设傻瓜套餐**：

- **Qwen 家族**：阿里云出品的开源模型，特别是 Coder (编程) 版本，很强。文档给了 32B（需单张高端显卡）、30B、480B（超级牛，需要8张顶级H200显卡）的例子。
- **GPT-OSS**：一种特殊架构开源模型。
- **GLM 家族**：清华推出的智谱大模型 GLM-4.5，甚至提到了带有类似 O1 会“深度思考”的版本。
- **自定义 (**`--vllm`**)**：如果你想跑的冷门模型不在内，可以用 `--vllm` 自己把底层参数传过去（比如限制张量并行度等）。

## 8 & 9. DataCrunch Setup / RunPod Setup (云平台详细教程)

> **新手解析**：这是实操指南。手把手教你在目前主流的两个便宜买显卡算力的网站（DataCrunch 和 RunPod）上：

1. 先买一块硬盘（网络卷），
2. 再买 GPU 机器并且把硬盘挂在上面，
3. 最后用 `pi pods setup` 把那段官方给的连接口令贴进去。系统会自动把你模型存到这里，避免下次开机还要重下几十个 GB。

## 10. Multi-GPU Support (多显卡支持)

> **新手解析**：如果你家里有矿/公司有钱，买了一台带 4 张 或 8 张显卡的机器：

- **互不干扰**：跑 3 个小模型，它会自动扔到 GPU 0、GPU 1、GPU 2 上，不会大家挤在一张卡上。
- **大力出奇迹**：如果你要跑拥有惊艳效果的超级大模型（如 70B 或 480B 参数），一张卡塞不下，就可以用 `--tensor-parallel-size` 强行把这个模型拆成几份，分布到所有的卡上一起跑（算力并联）。

## 12. Standalone Agent CLI (独立命令行小助手)

> **新手解析**：
> 这相当于一个独立小玩具 `pi-agent`。就算你不用这个程序管云服务器，你也可以只用这个命令行机器人去随意连本机的模型，或者直接输入 OpenAI 的 API Key 去直接调商业大模型。
> 功能很完善：支持记录以前聊的内容，支持回答代码时给你带上好看的颜色（语法高亮），包含文件查询工具（能帮你找代码）。

## 13. Tool Calling Support (工具调用的底层细节)

> **新手解析**：因为各大公司做的模型，调用的工具系统格式（解析器）千奇百怪。这段是告诉进阶玩家，`pi` 在背后帮你做了什么脏活累活。例如为 Qwen 强行绑定了 `hermes` 格式解析，帮 GLM 绑定了专门的 `glm4_moe`。如果你的小模型不支持工具，可以加上参数禁用掉（`--disable-tool-call-parser`）。

---

## 14. Memory and Context Management (显存和上下文怎么管)

> **新手解析**：这是底层 VLLM 必须要懂的核心概念，`pi` 让它变简单了：

- **Memory (显存)**：决定它占据显卡的统治力。你想让一百个人同时访问你的模型，就把 memory 设低（比如 30%），你想让模型拥有惊人记忆能力一个人独享，就设 90%。
- **Context (上下文)**：决定一段对话能塞进多少字的文本。4k 也就是几千字，128k 就能一次读一整本小书了。

---

## 15 & 16. Session & Event Architecture (会话保存与事件流架构)

> **新手解析**：

- **存聊天记录**：加上 `-i` (Interactive) 进入聊天后，你的历史记录会保存在你电脑的 `~/.pi/sessions/` 文件夹下。加上 `-c` 就能继续上次话题。
- **架构设计**：这个工具底层用了事件流。意味着你要是开发个前端网页去接它，可以加上 `--json` 参数。终端吐出来的数据就会是一行行的 `{ type: "message"... }` 格式，对机器极为友好。

---

## 17. Troubleshooting (疑难杂症排除)

> **新手解析**：遇到问题的第一急救指南。

- **OOM (Out of Memory 内存溢出)**：卡死报错了怎么办？调小前面提到的 `--memory` 比例，或者缩短 `--context`。
- **怎么还没好 / 跑不起来**：用 `pi list` 看看它是不是活着。用 `pi ssh "nvidia-smi"` 查看云端显卡是不是真在工作。
- **下载模型被拒绝**：像著名的 Llama（Meta 出品）或 Mistral，人家是有版权的，你需要先去 HuggingFace 网站点个“同意协议申请”。

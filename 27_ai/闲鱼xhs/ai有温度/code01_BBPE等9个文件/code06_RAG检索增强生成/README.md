> 小红书 @AI有温度

1. 需要自行下载选择embedding模型对相应的包
2. 任何pdf文件均为加载解析
3. 本代码包含了RAG以下部分
    1. 文本分块
    2. 向量化
    3. bm25检索
    4. 向量检索
    5. 多路召回
    6. prompt工程
    7. RAG问答

    提供的BM25的实现并非工业级别，仅作为演示目的。在实际进行文本检索时，应该使用专业的文本检索引擎工具，例如Elasticsearch，以确保高效、可扩展和内存友好的实现
4. 待改进点：
    1. 文本块/表格/图片的结构解析分块
    2. 不同检索方法/模型的混合使用
    3. 检索倒排索引
    4. 多路召回规则
    5. 重排序模型

以上几点在RAG不同的业务中差异很大，**需要同学们在具体的业务中，在RAG的基本框架上优化改造整个流程**。

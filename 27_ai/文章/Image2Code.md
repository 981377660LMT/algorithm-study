- 如果只有 Image 情况下， VLM 会更有优势，如果有 Figma 稿子且结构清晰合理，用 Figma mcp 会有优势。
- B 端场景，figma schema 可以裁剪。原生的 figma mcp，返回了太多样式信息，但是 B 端一般都用组件库，自定义内容较少：figma schema 裁剪 + VLM < figma 原生 mcp。

提升 UI 的难点在于如何让大模型清晰地理解组件的具体类型。
通过对小模型进行微调，使其专注于组件识别任务。然而，过程中我们发现传统的视觉模型（如 YOLO）在处理视觉特征明显的实体组件时表现优秀，但对于虚拟组件（如表单或表格生成等）则存在较大的识别误差。基于此，我们提出了两套解决方案：一是让专用模型负责其擅长的任务，二是将复杂虚拟组件的识别交由经过微调的大模型处理，以实现分工协作。

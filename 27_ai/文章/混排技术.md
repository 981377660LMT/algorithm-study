- 混排协议：采用 Markdown（AI 友好，大厂通用）+ HTML 标签扩展（稳定性更强）
  Markdown 支持两种扩展方式：

  - 语法扩展：通过自定义的特殊符号组合方式，实现扩展（如 :::custom-component:::）。缺点明显，AI 在「公域知识学不到」该语法，需通过说明进行强制约束，「易产生幻觉」
  - HTML 标签扩展（选择）：使用自定义标签名（如<custom-component>），在渲染时转换成对应的 react 组件。这类知识在公域丰富，AI 的 HTML 编码能力是很强的，稳定性更好

- 渲染容器：以 React Markdown 为基座，支持流式渲染（通过正则/AST 修复标题、链接、JSON 等内容跳变问题）
- 可观测性：全链路行为上报（SSE 发起 → 正文完成）+ 统一看板，实现端到端观测

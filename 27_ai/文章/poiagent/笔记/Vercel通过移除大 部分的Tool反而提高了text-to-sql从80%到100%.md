# https://vercel.com/blog/we-removed-80-percent-of-our-agents-tools

以下是 Vercel 博客文章《我们删除了 Agent 80% 的工具》的精简讲解：

### 核心观点

放弃复杂的定制化 Agent 架构，化繁为简。仅赋予大模型**执行原生 Bash 命令**的能力（即“文件系统 Agent”），即可大幅提升 AI 复杂任务的成功率、运行性能并降低开发维护成本。

### 背景与痛点

Vercel 构建了一个将自然语言转为 SQL 的内部 AI 助手（d0）。最初，团队为其开发了大量的专用工具（如：查表、查询验证、错误恢复等），配置了复杂的 Prompt 限制，试图“替模型思考”并防止其出错。

- **代价**：系统变得极其脆弱、运行缓慢，需要耗费大量时间维护逻辑脚手架。

### 解决方案：极简的“文件系统 Agent”

团队删除了 80% 的特定动作工具，转而采用一种极简单粗暴的方法：

- **唯一工具**：在沙箱内执行任意的终端 Bash 命令。
- **数据存储**：把数据库表结构和语义层资料，写成规范的目录和文本文件（YAML、Markdown 等）。
- **工作方式**：大模型像一名人类程序员一样，直接使用经典的 Unix 原生命令（如 `ls`, `cat`, `grep`, `find`）来自主浏览文件、搜索信息、建立心智模型，并最终编写 SQL。

### 对比成效显著

相比满是限制和专属工具的旧版本，极简新版带来的提升包括：

- **速度提升 3.5 倍**（平均执行 274 秒降至 77 秒）。
- **成功率从 80% 达至 100%**。
- **Token 消耗锐减 37%**。
- **所需推理步骤减少 42%**。

### 给开发者的关键启示

1. **不要与趋势对抗（做减法）**：大模型本身具备极强的推理能力。每多加一个专用工具，都是在限制它；放开手脚，让它自己决定如何调用系统命令，效果往往更好。
2. **复用经典抽象**：Unix 文件系统和 `grep` 是历经了 50 年考验的优秀工具，无需用复杂的 Agent API 为检索和浏览重复造轮子。
3. **基础设施决胜**：这种极简架构能成功的前提，是底层数据层本身具有良好、结构化的文档与命名规范。如果原始数据一团糟，开放文件权限也无济于事。

# Code execution with MCP: Building more efficient agents

Tool -> MCP -> Skill

**skill前身：渐进式加载+多用代码执行**

以下是对 Anthropic 文章《通过代码执行结合 MCP 构建更高效 Agent（Code execution with MCP）》的精简讲解：

### 核心观点

让 AI Agent **通过编写代码来调用 MCP 工具**，而不是在对话上下文中直接调用。这样可以大幅减少 Token 消耗，降低延迟，并提升复杂任务的处理能力。

### 痛点：传统直接调用工具的缺陷

随着连接的 MCP 工具增多，大模型面临两大 Token 消耗难题：

1. **上下文过载**：大模型需要预先加载所有工具的定义和参数说明，极其耗费 Token。
2. **中间结果冗余**：工具返回的巨大中间结果（如长篇会议记录、数万行表格）必须全部回传到大模型上下文中，才能作为下一步操作的输入，成本高昂且容易导致模型出错。

### 解决方案：MCP 沙箱编程 (Code Execution)

为 Agent 提供一个代码执行环境（沙箱）。将 MCP 工具封装成可以引用的代码 API（如模拟成文件系统中的库）。让大模型自己写脚本（如 TypeScript）来拉取数据、处理逻辑，最后返回结果。

### 核心优势

1. **按需加载 (节省 98% Token)**：大模型可以像浏览文件系统一样，只探索并加载当前任务所需的特定工具定义。
2. **在沙箱内处理中间数据**：通过代码直接处理大数据（如通过代码过滤出 10000 行图表中的 5 行关键数据），**仅将最终小结果**呈现给大模型。
3. **更强的控制流**：利用纯代码的 `while`/`if` 循环处理重试或轮询，避免在聊天界面与大模型反复来回对话产生延迟。
4. **强隐私保护**：敏感数据（如用户邮箱、电话等 PII 隐私）可以直接在代码环境中流转（比如从云盘直接传到数据库），无需暴露给大模型的上下文窗口；此外，配合客户端拦截可以实现自动脱敏。
5. **技能留存与记忆**：Agent 可以将查询到的结果保存成文件（如 CSV），甚至可以将自己写好的这套“工作流代码”保存下来，沉淀为以后可以反复调用的“专属技能 (Skills)”。

### 总结与代价

用“代码执行”取代“直接调用”虽然增加了沙箱基建的安全与运维成本，但这是解决 Agent 在海量工具和复杂任务中扩展瓶颈的最佳软件工程实践方案。

- 为了解决“高智商、低经验”的矛盾，Anthropic 提出了一个关键概念：Agent Skills。
- 先让模型“懂业务”，再谈多智能体架构：不要迷恋复杂的 Agent 架构，先把精力花在提炼和封装具体的专业知识（Skills）上。

---

对正在做 LLM / Agent 的团队来说，这至少意味着三件事：

1. 在设计任何 Agent 时，先问：

- 这个场景里有哪些东西可以沉淀成 Skill？
- 这些 Skill 的维护人是谁？
- 我们有没有一套流程来迭代 Skills，而不是只改 prompt？

2. 在工程实践上，优先把业务逻辑放进 Skills：

- 统一通过目录 + SKILL.md + 附属资源组织；
- 让 Agent 成为“技能调度器”，而不是“所有逻辑的大杂烩”。

3. 在组织层面，把 “技能库” 当成长期资产：

- 新员工、外包团队、自动化 Agent 都可以在同一套 Skills 上对齐行为；
- 知识积累不再只是“存在某个人脑子里”或“散落在 Confluence 文档里”。
  当你有了一套不断进化的技能库，一个通用大模型就可以通过“加载不同 Skills”：
- 上一秒是文档分析师；
- 下一秒变成税务专家；
- 再下一秒切成代码审查员或运营分析师。
  先让模型真正“懂业务”，再考虑要不要用更多智能体、更复杂的编排。
  从“先造 Agent”到“先造 Skills”，是让 Agent 项目更务实、更可持续的一条路线。

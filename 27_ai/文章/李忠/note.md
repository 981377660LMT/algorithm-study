- [Chrome Extension 是 Vibe Coding 的绝佳载体](https://limboy.me/posts/chrome-extension-vibe-coding-container)
  Chrome Extension 简直是 Vibe Coding 的绝佳载体。因为它的反馈回路短，工程负担轻，能力还很强（读写本地文件、存储、网络请求、自定义网页内容等等），相比于开发 Native App，Chrome Extension 不需要配置繁琐的编译环境，也不涉及复杂的系统级 API。它本质上就是网页，使用着 AI 最擅长的 HTML、CSS 和 JavaScript。即使是一个没有编程经验的人，只要花一些时间熟悉工具，也能快速上手。
  **Grease monkey script 也是同理**

- [Coding Agent 时代，App 的核心竞争力是什么？](https://limboy.me/posts/coding-agent-app-core-compentencies)
  `AI 无法生成、无法复制且具有时间复利 的东西`
  - 从功能实现转向用户洞察
    代码是可以被复制的显性知识，但关于「为什么要这样做」的隐性知识是 AI 难以窃取的。
  - 数据资产与迁移壁垒
    产品的设计逻辑应从「提供工具」转向「沉淀资产」。让产品越用越懂用户，这种基于数据的个性化体验，是冷冰冰的 AI 克隆版无法比拟的。
    > 一个典型的例子，就是苹果的产品，虽然 iOS 26 和 macOS Tahoe 被大量吐槽，但下决心转向其他阵营的还是少数，因为迁移成本太高了。

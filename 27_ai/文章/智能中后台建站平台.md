# 智能中后台建站平台

- 技术实现
  - 快速执行
    - 生成策略
      Agent 采用 Workflow 的实现方式，生成流程：生成页面大纲 -> 页面 UI 生成(组件布局、组件属性填充) -> 页面请求(HTTP、RPC 请求) -> 页面逻辑(事件、表达式)，在页面大纲、页面 UI、页面事件这几个环节进行了 SFT 训练。
    - 生成策略
      数据质量和多样性远大于数据数量
    - 数据工程
    - 模型训练
    - 效果评测
  - 自主规划
    - 技术流程
      Agent 采用 PlanAct 的实现方式，整体流程是：用户输入 -> Planner 动态规划 -> 任务派发给 Agent 专家 -> 调用工具执行 -> 任务反思 -> 完成任务
    - 生成策略
      SandBox 环境接入
    - 模型训练

---

- 多 Agent 架构：Agent 采用 PlanAct 的实现方式，整体流程是用户输入 -> Planner 动态规划 -> 任务派发给 Agent 专家 -> 调用工具执行 -> 任务反思 -> 完成任务
- 在线文档搜索：在任务规划前会通过 SearchAgent 动态搜索业务知识，包括文档、云、业务 Wiki 空间等知识库，生成结果和业务需求相关性更高
- 评估反思：任务完成后 会执行 UT/E2E 测试用例发现问题并自我迭代，包括页面 UI、交互逻辑、功能完整性等
- 沙盒环境：对接 DevContainer 沙箱，保证 Agent 稳定的执行环境，支持任务长时间执行；沙盒里使用 Browser_use 技术支持站点复刻能力
- 底层架构：在工具调用、上下文压缩策略、提示词、记忆、安全防护等维度对齐 Claude Code

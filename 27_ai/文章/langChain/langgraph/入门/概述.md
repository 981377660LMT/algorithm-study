这份文档介绍了 **LangGraph**，这是一个用于构建、管理和部署长期运行、有状态 Agent 的低级编排框架。

以下是核心内容总结：

### 1. 什么是 LangGraph？
*   **定位**：一个非常底层的框架，专注于 Agent 的**编排 (Orchestration)**。
*   **用途**：构建复杂的、有状态的、需要长期运行的 Agent 工作流。
*   **与 LangChain 的关系**：LangGraph 可以独立使用，不强制依赖 LangChain。LangChain 提供了高级的 Agent 抽象（预构建架构），而 LangGraph 提供了底层的控制能力。

### 2. 快速开始

**安装**
```bash
npm install @langchain/langgraph @langchain/core
```

**Hello World 示例**
这是一个最简单的图结构：开始 -> 调用模拟 LLM -> 结束。

```typescript
import { MessagesAnnotation, StateGraph, START, END } from "@langchain/langgraph";

// 定义节点功能
const mockLlm = (state: typeof MessagesAnnotation.State) => {
  return { messages: [{ role: "ai", content: "hello world" }] };
};

// 构建图
const graph = new StateGraph(MessagesAnnotation)
  .addNode("mock_llm", mockLlm)
  .addEdge(START, "mock_llm")
  .addEdge("mock_llm", END)
  .compile();

// 运行图
await graph.invoke({ messages: [{ role: "user", content: "hi!" }] });
```

### 3. 核心优势
LangGraph 为构建生产级 Agent 提供了关键的基础设施支持：
*   **持久化执行 (Durable Execution)**：支持断点续传，即使发生故障也能恢复，适合长期运行的任务。
*   **人机协同 (Human-in-the-loop)**：允许人工在执行过程中检查、修改 Agent 的状态或批准操作。
*   **综合记忆 (Comprehensive Memory)**：支持短期工作记忆（用于当前推理）和长期记忆（跨会话）。
*   **调试与可观测性**：与 **LangSmith** 深度集成，提供执行路径追踪和状态转换的可视化。
*   **生产级部署**：专为处理有状态工作流的扩展性挑战而设计。

### 4. 生态系统
LangGraph 可以与以下工具配合使用以增强开发体验：
*   **LangSmith**：用于追踪、评估和监控。
*   **LangGraph Platform**：用于部署和扩展 Agent 的专用平台。
*   **LangChain**：提供模型集成和组件库。

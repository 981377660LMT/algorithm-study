这段视频是对机器学习（Machine Learning, ML）核心概念的快速综述。以下是根据视频内容整理的术语解释清单，按照逻辑分类进行了组织：

### 1. 基础概念

- **人工智能 (Artificial Intelligence, AI)**：机器执行通常需要人类智能的任务（如决策、识别图像）的能力。并非所有 AI 都是机器学习（例如基于规则的系统）。
- **机器学习 (Machine Learning, ML)**：AI 的一个分支，使计算机能够从数据中学习并改进，无需针对特定任务进行显式编程。
- **算法 (Algorithm)**：计算机解决问题的一组定义明确的指令或规则（例如用于导航的 Dijkstra 算法或烹饪食谱）。
- **模型 (Model)**：一种经过“训练”的数学表示，用于识别数据中的模式并做出预测（例如线性回归线）。

### 2. 数据相关术语

- **数据 (Data)**：用于分析的信息（数字、文本、图像像素等）。
- **实例 (Instance/Sample)**：单个完整的数据单元（例如表格中的一行）。
- **特征 (Feature)**：用于预测的输入变量（例如房子的面积、卧室数量）。
- **特征工程 (Feature Engineering)**：利用领域知识从原始数据中创建更能代表问题的新特征。
- **特征缩放 (Feature Scaling)**：将不同范围的数值特征标准化到同一尺度（如 0 到 1），这对梯度下降等算法至关重要。
- **维度 (Dimensionality)**：数据集中特征的数量。
- **目标 (Target/Label)**：模型试图预测的变量（例如实际房价、是否为垃圾邮件）。
- **训练数据 (Training Data)**：用于教导模型的数据集，包含输入和正确答案。
- **测试数据 (Test Data)**：以前未见过的数据，用于最终评估模型性能，严禁用于训练。
- **数据泄露 (Data Leakage)**：测试数据意外混入训练过程，导致评估结果虚高。

### 3. 先进的学习范式

- **监督学习 (Supervised Learning)**：使用带标签的数据（有正确答案）进行训练。占比约 70%。
- **无监督学习 (Unsupervised Learning)**：使用无标签数据，让算法自行发现结构或模式（如聚类）。
- **强化学习 (Reinforcement Learning)**：通过试错学习，根据奖励和惩罚机制进行优化（如 AlphaGo）。

### 4. 模型性能与复杂度

- **模型拟合 (Model Fitting)**：调整模型参数以匹配数据的过程。
- **过拟合 (Overfitting)**：模型太复杂，记住了训练数据中的噪声而非规律，导致在从未见过的数据上表现差（高方差）。
- **欠拟合 (Underfitting)**：模型太简单，无法捕捉数据中的基本模式（高偏差）。
- **偏差 (Bias)**：模型假设的简化程度。高偏差意味着模型假设过于简单（如用直线拟合曲线）。
- **方差 (Variance)**：模型对训练数据微小变化的敏感度。高方差通常意味着过拟合。
- **偏差-方差权衡 (Bias-Variance Tradeoff)**：寻找模型复杂度的平衡点，以同时最小化偏差和方差。
- **噪声 (Noise)**：数据中不代表真实模式的随机误差。

### 5. 训练机制与优化

- **参数 (Parameters)**：模型在训练过程中内部学习到的值（如线性回归的斜率和截距，神经网络的权重和偏置）。
- **超参数 (Hyperpathameters)**：在训练开始前人为设置的配置（如学习率、层数），用于控制学习过程。
- **损失函数/成本函数 (Cost/Loss Function)**：衡量模型预测值与真实值差距的指标（如均方误差 MSE）。训练的目标是最小化它。
- **梯度下降 (Gradient Descent)**：一种优化算法，通过迭代调整参数来最小化损失函数。
- **学习率 (Learning Rate)**：梯度下降中每一步更新幅度的超参数。太大可能无法收敛，太小则训练过慢。
- **正则化 (Regularization)**：通过惩罚模型复杂度来防止过拟合的技术。
- **Batch / Epoch / Iteration**：
  - **Batch**: 一次处理的一组数据样本。
  - **Iteration**: 处理完一个 Batch 的过程。
  - **Epoch**: 完整遍历一次整个训练数据集的过程。

### 6. 评估

- **验证 (Validation)**：在开发过程中使用一部分数据调整超参数和选择模型。
- **交叉验证 (Cross-Validation)**：多次分割数据进行训练和验证，以获得更稳健的性能评估。
- **评估 (Evaluation)**：使用测试集和特定指标（准确率、精确率、召回率等）衡量最终模型的表现。

这门课从工业界的视角讲解现代搜索引擎技术。这是第一节课，解释一些搜索引擎的基本概念，包括查询词 (query)、文档 (document)、查询建议、搜索结果页、曝光、点击、点击率、有点比、通用搜索（通搜）、垂直搜索（垂搜）。

---

基于王树森教授的《搜索引擎技术》第一讲视频教程，以下是对搜索引擎基本概念、核心指标及架构分类的深度分析与讲解。

### 1. 课程定位与技术背景

本课程聚焦于**工业界实战技术**，而非纯学术理论。

- **技术栈基础**：现代搜索引擎已全面拥抱深度学习（Deep Learning）和自然语言处理（NLP）。课程明确指出，如果缺乏这两个领域的知识，将难以理解现代搜索架构。
- **与推荐系统的关系**：搜索（Search）与推荐（Recommendation）在`底层技术`上有大量共通之处（“搜推广”不分家），但在`目标极其评价指标`上存在本质差异。

### 2. 用户交互与基本术语 (Terminology)

搜索引擎的交互链路看似简单，实则包含复杂的工程逻辑：

- **Query (查询词)**：用户的显式输入。
- **SUG (Suggestion/查询建议)**：
  - **功能**：在用户输入过程中自动补全。
  - **洞察**：不仅是为了节省用户时间，更是为了**规范化查询意图**。通过引导用户点击标准的 SUG 词，可以减少长尾、拼写错误或模糊的 Query，降低后端检索难度。
- **Document (文档)**：检索的实体。在不同场景下可是网页、商品、视频或本地生活店铺。
- **SERP (Search Engine Results Page)**：搜索结果页。
  - **曝光模式**：单列（Google/百度）vs 双列（淘宝/小红书）。
  - **工程视角**：前端形态（单列或双列）仅影响 UI 和交互数据分布，**不影响后台核心算法与架构**。

### 3. 核心评价指标的深度辨析 (Metrics)

这是本节课最核心的概念，理解这些指标主要区分**搜索**与**推荐**的不同优化目标。

#### A. 文档点击率 (Document CTR)

- **定义**：$\frac{\text{文档点击总次数}}{\text{文档曝光总次数}}$
- **特征**：数值通常较低（约 10%）。
- **逻辑**：独立看待每一个 Item 的吸引力。这通常是**推荐系统**的核心指标（因为推荐追求用户过度消费内容）。

#### B. 查询词点击率 / 有点比 (Query CTR / UCTR)

- **定义**：$\frac{\text{点击的总搜索次数}}{\text{总搜索次数}}$
- **特征**：数值较高（约 70%）。
- **逻辑**：以**Session（会话）**或**Query** 为单位。只要用户在一次搜索中点击了 _任意_ 一个结果，即视为该次搜索“成功”。
- **洞察**：**有点比是搜索引擎最重要的指标**。
  - **原因**：搜索是工具导向的（Utility-oriented）。用户的目的是“找到答案”。如果用户搜索了一次，给出的 10 个结果中他点击了 1 个，他的需求就必须得到了满足。反之，如果在推荐系统中，用户只点了一个，可能意味着用户粘性不够。

#### C. 首屏有点比 (First-Screen UCTR)

- **定义**：$\frac{\text{点击首屏结果的搜索次数}}{\text{总搜索次数}}$
- **特征**：数值略低于有点比（约 60%）。
- **洞察**：这是衡量**排序质量（Ranking Quality）**的关键指标。如果有点比高但首屏有点比低，说明系统召回了相关文档，但排序算法（Ranking Model）很差，没把用户想要的排在前面，迫使用户下滑寻找。

### 4. 垂直搜索 vs. 通用搜索 (Vertical vs. General)

两者不仅仅是业务领域的区别，本质是**数据结构化程度**与**意图识别难度**的区别。

| 特性         | 垂直搜索 (Vertical Search)                           | 通用搜索 (General Search)                                      |
| :----------- | :--------------------------------------------------- | :------------------------------------------------------------- |
| **代表应用** | 淘宝(电商)、知网(学术)、美团(本地生活)               | Google、百度、抖音(综合)                                       |
| **文档结构** | **高度结构化**。商品有价格、类目；论文有作者、年份。 | **非结构化/杂乱**。网页文本、UGC 视频、混合多模态内容。        |
| **检索方式** | 易于进行 Database 风格的字段过滤 (Filtering)。       | 需要复杂的 NLP 技术提取信息、建立倒排索引。                    |
| **用户意图** | **意图明确**。搜“苹果”一定是买手机或水果。           | **意图多义**。搜“苹果”可能是看发布会视频、买手机、还是查百科？ |
| **技术难度** | 相对较低，侧重于特定业务逻辑。                       | **极高**。需处理多意图识别、非结构化抽取、跨模态排序。         |

### 5. 课程学习路线与实战建议

- **预备知识**：建议先学习推荐系统（技术相对简单，重合度高）以及 NLP 基础。
- **核心链路预告**：
  1.  **查询词处理 (Query Processing)**：意图识别、纠错、改写。
  2.  **召回 (Recall)**：快速从海量数据中捞出候选集。
  3.  **排序 (Ranking)**：精细化计算相关性与点击率。
  4.  **查询词推荐**：被动搜索技术。

**总结**：这一讲不仅建立了搜索的术语体系，更重要的是指出了搜索优化的北极星指标（有点比）以及通用搜索面临的“`非结构化数据`”与“`模糊意图`”两大核心挑战。

# 优化 CI 构建时间

链接：https://code.visualstudio.com/blogs/2020/02/10/ci-speed

## 深入分析

这篇文章不是关于VS Code产品功能，而是VS Code**团队自身的CI/CD优化实践**。

### 问题
- VS Code的构建时间日益增长，严重影响了开发效率
- 当时完整构建需要10-15分钟，这在敏捷开发中不可接受
- 开发者的"反馈循环"变长，降低了生产力

### 优化方向
1. **增量构建** - 只编译改动的部分，而非全量重编
2. **并行构建** - 充分利用多核CPU
3. **缓存策略** - 利用Azure Pipelines的缓存层
4. **代码优化** - 减少冗余依赖，优化bundle大小

### 对用户的影响
- 虽然是内部优化，但能加快特性迭代速度，最终受益的是用户
- 更快的bug修复周期，更高的发布频率

### 工程文化
- VS Code团队定期分享内部最佳实践，这是开源精神的体现
- 这篇文章帮助了许多其他项目团队优化CI/CD

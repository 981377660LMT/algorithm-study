# 使用平分法解决扩展问题

链接：https://code.visualstudio.com/blogs/2021/02/16/extension-bisect

## 一针见血的分析

1. **二分查找解决工程痛点**：对于拥有上百个插件的用户，手动定位问题源就像大海捞针。将 Git 的 `bisect` 思想引入插件管理，将原本 $O(N)$ 的排查复杂度降低到 $O(\log N)$。
2. **防错与隔离策略**：Bisect 的第一步是禁用所有插件。这不仅是为了开始二分，更是为了验证问题是否由 VS Code 本身引起的（防误报）。
3. **低成本排错生态**：通过自动化「禁用-重载-反馈」循环，VS Code 鼓励用户积极上报插件 Bug，从而间接维护了整个插件市场的质量。

## 摘要

VS Code 引入了 **Extension Bisect** 实用工具，帮助用户快速定位导致异常（如崩溃、性能下降或意外行为）的「坏」插件。

- **工作机制**：通过不断将启用插件的数量减半，在有限的步骤内（例如 24 个插件只需约 5 步）精准定位问题。
- **操作流程**：通过 `Help: Start Extension Bisect` 命令启动，用户根据排查结果选择 `Good now` 或 `This is bad`。
- **纠错保护**：首回合会尝试完全不加载插件，确保问题确实是由插件而非核心软件引起的。
- **用户收益**：无需手动逐一开关插件，极大缩短了排错时间。

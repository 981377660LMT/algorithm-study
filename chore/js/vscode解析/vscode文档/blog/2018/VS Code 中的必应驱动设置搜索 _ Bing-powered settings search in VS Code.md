# VS Code 中的必应驱动设置搜索

链接：https://code.visualstudio.com/blogs/2018/06/20/settings-search

## 深入分析

这个功能看似小，但代表了一个有趣的设计哲学——如何帮助用户发现隐藏的功能？

### 问题背景

- VS Code有数百个设置项，用户通常只知道5-10个
- 许多强大的功能被埋在菜单或设置中，新用户很难发现
- 官方文档很完善，但没有人会逐个阅读

### 解决方案

1. **自然语言搜索** - 用户可以输入"我想自动保存"，系统会找到auto-save相关的设置
2. **Bing AI的引入** - 使用Microsoft的自然语言理解能力，提高搜索准确性
3. **上下文推荐** - 根据用户的语言和当前编辑的语言，推荐相关设置

### 为什么提到"必应"？

- 这是微软的云计算能力的展示
- 但这篇文章的反响并不热烈，说明"AI辅助设置搜索"在当时还不是刚需
- 直到ChatGPT出现后，这种功能的价值才被重新认识

### 设计学启示

- 即使产品功能完整，如果用户发现不了，也白搭
- VS Code的"设置搜索"解决的是一个元问题——帮助用户探索产品能力
- 这种"教用户"的思维，反映了成熟的产品团队

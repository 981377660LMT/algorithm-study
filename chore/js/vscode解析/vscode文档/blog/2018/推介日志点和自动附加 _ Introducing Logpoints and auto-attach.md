# 推介日志点和自动附加

链接：https://code.visualstudio.com/blogs/2018/07/12/introducing-logpoints-and-auto-attach

## 一针见血的分析

这篇文章深刻地洞察了现代开发者（特别是 JS/Node 开发者）的调试心理与习惯：

1.  **“console.log 调试法”的正名**：团队敏锐地观察到，大量开发者（特别是 JS 领域）依然偏爱 `console.log`。原因在于传统断点会阻断执行流，而开发者往往只是想观察状态的变化。**Logpoints（日志点）** 的引入，将“动态注入日志”与“无需停止运行”结合，解决了调试后忘记删日志的尴尬，是“非侵入式调试”的里程碑。
2.  **降低调试的认知摩擦**：VS Code 意识到 `launch.json` 的配置过重。**Auto Attach（自动附加）** 功能的出现，让开发者可以在终端里随手敲下 `node --inspect`，编辑器就能自动感知并连上调试器。这种“感知意图”而非“强制配置”的设计，极大地降低了调试功能的门槛。
3.  **Launch vs Attach 的哲学辩论**：文章理清了两个核心概念：Launch 是“我想在调试环境下启动你”；Attach 是“你已经在跑了，我想连上去看看”。VS Code 最终选择了两者并重，但通过 **NPM Scripts Explorer** 等 UI 引导，让整个过程变得更加可视化。
4.  **DAP 协议的普适性**：Logpoints 的成功不仅仅属于 Node.js。通过 Debug Adapter Protocol (DAP) 的扩展，Python、Chrome、Java 等调试器迅速跟进。这证明了只要协议定义得当，一个微小的交互创新可以迅速惠及整个多语言生态。

## 摘要

2018年7月，VS Code 团队分享了在调试领域的数项重大创新。针对开发者更倾向于记录日志而非中断程序的习惯，引入了不中断运行的“日志点（Logpoints）”。同时，为了简化复杂的调试配置，推出了“自动附加（Auto Attach）”功能，能够自动捕捉终端中启动的调试进程。这些改进旨在降低调试工具的使用门槛，使其更贴合开发者的自然工作流。

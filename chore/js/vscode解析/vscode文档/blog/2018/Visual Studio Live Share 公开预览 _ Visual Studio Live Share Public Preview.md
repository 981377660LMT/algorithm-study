# Visual Studio Live Share 公开预览

链接：https://code.visualstudio.com/blogs/2018/05/07/live-share-public-preview

## 一针见血的分析

这篇文章展现了 Live Share 从实验性功能向生产级工具进化的关键改进：

1.  **共享终端的“最后一块拼图”**：在本地开发中，命令行工具（构建、测试、数据库迁移）是不可或缺的。Live Share 公开预览版引入了 **共享终端 (Shared Terminals)**。Host 可以共享一个只读终端，甚至给 Guest 写入权限。这种深层环境权限的共享，让“远程环境配置协作”变得真正可行。
2.  **安全性与便利性的平衡**：共享本地资源总伴随着风险。Live Share 引入了基于 `.gitignore` 的排除机制（`.vsls.json`），并增加了 Guest 加入时的显式审核。这标志着协作模式从“社区尝鲜”转向了“企业合规”。
3.  **多语言的普适性**：文章强调了它是“跨语言、跨平台”的。无论你是在调 Python Web 应用还是 Go 微服务，Live Share 都能通过协议转发语言服务的元数据。Guest 无需安装任何语言扩展或运行时，这种“瘦客户端”式的体验极大地降低了协作门槛。
4.  **云原生协作的预演**：Live Share 实质上是将“计算”与“表现”完全分离的架构在多人场景下的极致应用。它奠定了微软后来一系列“开发环境作为一种服务”的产品哲学。

## 摘要

2018年5月，Visual Studio Live Share 进入公开预览阶段。除了核心的代码共享和调试功能外，新增了共享终端、文件排除策略和访客加入审核等关键功能。这一切旨在提供一种无需环境配置、安全且全功能的远程实时协作体验，目前已支持 Python、Go、React Native 等多种语言和框架。

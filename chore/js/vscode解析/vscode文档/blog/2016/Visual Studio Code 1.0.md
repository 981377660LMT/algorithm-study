## 深入分析

VS Code 1.0 的发布是开发者工具史上的一个里程碑，标志着从一个“实验项目”正式演变为一个支撑百万级用户的成熟平台。

1.  **“Monaco” 的起源与使命**：
    - 起源于微软内部代号为 “Monaco” 的项目，最初旨在回答一个问题：“我们能否构建一个基于浏览器的编辑器，且快到不像是在浏览器里打字？”。
    - **架构选择**：最终选择了 GitHub 的 Electron 壳，结合了 Web 技术的灵活性（UI 迭代快）和 Node.js 的能力（直接操作系统资源）。
2.  **“吃自己的狗粮 (Dogfooding)” 哲学**：
    - VS Code 指导原则：团队必须使用自己开发的工具来开发工具。
    - **内置功能的权衡**：核心关注“编辑器常用任务”——导航、调试和 Git，其他功能则交给插件系统。有趣的是，内置的 JS/TS 支持其实也是通过同样的插件 API 实现的。
3.  **社区与开源的转型**：
    - 在发布 1.0 之前的几个月，VS Code 毅然决定开源并转向透明开发。这一决定直接导致了扩展数量的爆炸式增长（发布时已超过 1000 个），使其迅速超越了传统的轻量级编辑器。

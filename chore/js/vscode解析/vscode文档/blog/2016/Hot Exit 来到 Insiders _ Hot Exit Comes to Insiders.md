# Hot Exit 来到 Insiders

链接：https://code.visualstudio.com/blogs/2016/11/30/hot-exit-in-insiders

## 一针见血的分析

这篇文章介绍了提升用户“安全感”的核心功能：

1.  **数据的极致安全性**：开发者最恐惧的事情就是编辑器崩溃或误关导致的代码丢失。“Hot Exit”通过周期性的静默备份，确保用户可以在不保存文件的情况下直接关闭编辑器，并在下次启动时无缝恢复。这不仅是一个功能，更是一种对用户工作流的资产保护。
2.  **优雅的交互权衡**：团队最初考虑将备份与特定文件夹绑定，但由于担心用户忘记备份文件所在的文件夹，最终决定将其提升到**应用范围级（Application Level）**。这意味着当你再次启动 VS Code 时，它会自动拉起所有包含未保存更改的工作区。
3.  **开发效率的倍增器**：一个有趣的细节是，Hot Exit 解决了“安装插件需重启”时的弹窗骚扰。因为有了自动备份，VS Code 可以直接 reload 窗体而无需询问用户是否保存，这极大地平滑了插件生态的使用体验。
4.  **底层文件系统的巧妙利用**：Hot Exit 的实现并不依赖复杂的数据库，而是利用本地文件系统的特定 Backup 路径（通常在 `UserData/Backups` 下）。这种简单、鲁棒的设计原则是桌面软件稳定性的基石。

## 摘要

2016年11月，VS Code 在 Insiders 版本中引入了 Hot Exit 功能。该功能会自动备份所有未保存的文件内容，允许用户直接退出应用而无需手动保存。这不仅提供了意外崩溃后的数据保护，还让安装扩展后的重新加载过程变得更加顺滑。该功能在 1.8 版本正式引入 Stable 分支。

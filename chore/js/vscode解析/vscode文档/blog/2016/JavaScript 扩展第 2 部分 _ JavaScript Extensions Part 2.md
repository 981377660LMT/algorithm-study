# JavaScript 扩展第 2 部分

链接：https://code.visualstudio.com/blogs/2016/10/31/js_roundup_2

## 一针见血的分析

这篇文章揭示了 VS Code 如何通过“吞并”竞争对手的优势来迅速占领市场：

1.  **用户迁移的“投降式”优化**：VS Code 在这一阶段积极引入了 **Sublime Text 和 Atom 的 Keymaps**。这种做法极大地降低了老牌编辑器用户的迁移成本。它传递了一个信号：你无需学习新的工具，只需换一个更强的引擎。
2.  **Chrome Debugger 的常态化**：再次强调 Chrome 调试器，标志着“在编辑器内完成全生命周期开发”已成为官方安利的标准工作流。这种“不切屏”的诱惑对前端开发者是致命的。
3.  **格式化工具的标准化**：通过推介 **Beautify** 并支持 `.jsbeautifyrc`，VS Code 对齐了当时流行的代码风格管理方式。它不强求用户改用内置格式化，而是通过插件完美支持现有社区方案。
4.  **工作流自动化：Git Project Manager** 的提及说明 VS Code 团队不仅关注“写代码”，还关注“多项目管理”。这种对真实开发痛点的细微观察，助其在大规模项目协作中胜出。

## 摘要

2016年10月，JavaScript 扩展系列第二部分发布，重点推介了 Chrome 调试器、Git 项目管理工具、Beautify 以及面向 Sublime 和 Atom 用户的快捷键迁移包。这些工具旨在通过打通调试、格式化和多项目管理，进一步稳固 VS Code 作为 Web 开发首选编辑器的地位。

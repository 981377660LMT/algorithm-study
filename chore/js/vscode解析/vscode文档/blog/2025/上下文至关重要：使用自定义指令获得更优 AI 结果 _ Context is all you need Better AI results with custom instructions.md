# 上下文至关重要：使用自定义指令获得更优 AI 结果

链接：https://code.visualstudio.com/blogs/2025/03/26/custom-instructions

## 深入分析

### 1. 项目级 Context 的标准化

`.github/copilot-instructions.md` 的引入解决了团队开发中最大的痛点：如何让 AI 统一遵守特定的编码规范（如命名习惯、API 选型、架构分层）。通过这种“无痛注入”的方式，开发者不再需要在每个 Prompt 中重复“使用 2 空格缩进”、“遵循 ES6 标准”等繁琐指令。

### 2. 语气与逻辑的深度定制

文章展示了自定义指令不仅限于代码，还能改变模型的“性格”：

- **去礼貌化**：让模型停止无谓的道歉和由于过度谦虚导致的错误妥协。
- **场景化注入**：通过 `.prompt.md` 文件构建可复用的复杂 Prompts（如根据数据库 Schema 生成接口），实现了项目内部的“ Prompt 管理资产化”。

### 3. 低成本的调优模型

这是一种极其高效的“软微调（Soft Fine-tuning）”。开发者无需训练模型，只需维护几份 Markdown 文档，即可使通用大模型在特定项目语境下表现得像资深领域专家（Domain Expert）。

# 使用自带密钥扩展 VS Code 中的模型选择

链接：https://code.visualstudio.com/blogs/2025/10/22/bring-your-own-key

## 深入分析

### 1. 模型多元化：打破壁垒

"Bring Your Own Key" (BYOK) 的推出是 VS Code 历史上一个极其务实的决策。它标志着 Copilot 界面开始支持第三方模型服务商（典型的如 Anthropic 的 Claude 和 OpenAI 直接提供的 GPT 模型）。这不仅给了开发者更多的技术选择权，也让 VS Code 成为了一个更通用的 **AI 工具聚合窗口**。

### 2. 成本管理与企业采购

BYOK 特性直接回应了企业级用户的核心痛点：

- **成本透明化**：企业可以直接按自己的 Token 用量向模型供应商付费，而非绑定单一的 Copilot 订阅。
- **特定偏好**：不同的模型在特定任务（如写 Rust 或复杂逻辑推理）上互有优劣，允许自带 Key，实质上赋予了开发者随时切换“脑力引擎”的自由。

### 3. 战略转向

此举意味着微软不仅将 VS Code 视为自家 AI 服务的载体，也将其建设为一个开放的 AI 协议分发平台。这种开放性将进一步巩固 VS Code 在 AI 辅助开发时代的统治地位，使其免受单一模型迭代波动的影响。

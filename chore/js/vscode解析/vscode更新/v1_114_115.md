# VS Code v1.114 & v1.115 更新深入分析

> v1.114 发布日期: 2026-04-01 | v1.115 发布日期: 2026-04-08

这两个版本的核心主题非常明确：**全面拥抱 Agent-native 开发范式**。从 chat 体验优化到全新 Agents 伴侣应用，VS Code 正在从"编辑器 + AI 辅助"向"Agent 原生开发平台"快速演进。

---

## 一、v1.114：Chat 体验精细化 + 语义搜索统一

### 1.1 图片轮播支持视频预览

**设置项**: `imageCarousel.chat.enabled`, `imageCarousel.explorerContextMenu.enabled`

在 v1.113 引入图片轮播后，v1.114 将其扩展到视频。现在可以在 chat 附件或资源管理器右键菜单中直接播放和导航视频。

**实际意义**: 这对前端开发者非常有用。调试 UI 问题时，可以录制 bug 复现视频，直接附加到 chat 中让 Copilot 分析。不再需要用文字描述视觉问题——"一图/一视频胜千言"。

### 1.2 Copy Final Response（复制最终响应）

Agent 模式下，一次完整的回复包含：思考过程 → 工具调用链 → 最终答案。之前的"复制响应"会把所有中间过程都复制下来，非常冗长。

新增的 **Copy Final Response** 命令只复制最后一段 Markdown 结果（所有工具调用执行完之后的部分）。

**深度分析**: 这个看似简单的功能背后反映了一个重要趋势——agent 的"推理链"（思考 + 工具调用）和"最终输出"正在被有意识地分离。这种分离意味着:

- 用户可以把 agent 的最终结果直接粘贴到文档、PR description、Slack 中
- 中间推理过程仍然可见（用于审计和学习），但不会污染分享内容
- 这是 agent 交互模式成熟的标志：从"看 AI 怎么做"到"直接用 AI 的结果"

### 1.3 工作区搜索简化（Workspace Search Simplification）⭐ 重点

这是 v1.114 最重要的架构变更。

#### 变更前的问题

`#codebase` 工具之前的设计存在几个历史包袱：

1. **双重索引**: 同时维护"本地索引"和"远程索引"，本地索引仅支持几千个文件且不一定是语义搜索
2. **混合搜索**: `#codebase` 可能回退到模糊文本搜索（不准确、效率低）
3. **手动管理**: 用户需要自己理解和管理索引状态

#### 变更后的架构

```
旧架构:
┌─────────────┐     ┌─────────────┐
│ 本地索引     │     │ 远程索引     │
│ (非语义/有限) │     │ (语义/大规模) │
└──────┬──────┘     └──────┬──────┘
       │                   │
       └───────┬───────────┘
               │ #codebase（可能回退到模糊搜索）
               ▼

新架构:
┌─────────────────────────────────┐
│      统一语义索引                │
│  (自动构建、透明管理)            │
│  本地/远程存储对用户不可见        │
└───────────────┬─────────────────┘
                │ #codebase（纯语义搜索）
                ▼
┌─────────────────────────────────┐
│ Agent 其他搜索手段               │
│ (text grep, symbol search 等)   │
└─────────────────────────────────┘
```

**关键改变**:

| 维度     | 旧                   | 新                            |
| -------- | -------------------- | ----------------------------- |
| 索引类型 | 本地 + 远程          | 统一（是否已索引）            |
| 搜索类型 | 语义 + 模糊回退      | 纯语义                        |
| 管理方式 | 用户手动管理         | 自动按需构建                  |
| 大仓支持 | 远程索引支持百万文件 | 无 GitHub repo 的大仓暂不支持 |

**深度分析**: 这个变更的核心逻辑是——Agent 已经有了独立的 grep/text/symbol 搜索能力，不需要 `#codebase` 再去做低质量的回退搜索。让每个工具专注做好一件事：

- `#codebase` → 纯语义搜索（理解意图）
- Agent 自身 → 精确文本搜索、符号搜索（精确匹配）

**注意事项**: 之前用本地非语义索引的工作区需要重新索引。没有 GitHub repo 的超大代码库暂时可能无法索引。

### 1.4 Troubleshoot 支持历史会话

**设置项**: `github.copilot.chat.agentDebugLog.enabled`

`/troubleshoot` 命令现在可以通过 `#session` 引用任何历史 chat 会话进行诊断。

**实际使用场景**:

- "为什么我的自定义指令上次对话中被忽略了？" → `/troubleshoot #session`
- "昨天那个回复为什么特别慢？" → `/troubleshoot #session`
- 不需要复现问题，事后即可分析

这本质上是给 agent 的行为日志加了一个**可回溯的 debug 能力**。

### 1.5 TypeScript 6.0

VS Code 内置的 JS/TS 支持升级到 TypeScript 6.0。这是一个重大版本更新：

- 包含重要的修复和改进
- **废弃了一些旧选项**，为 TypeScript 7.0 的原生重写做准备

TypeScript 7.0 将是用 Go/Rust 等原生语言重写的版本，6.0 是过渡版本，开始清理历史债务。开发者应该关注自己项目中是否使用了即将废弃的 tsconfig 选项。

### 1.6 企业管理：禁用 Claude Agent

新增组策略 `Claude3PIntegration`（bool），管理员可以禁止组织内用户使用 Claude agent。

这反映了企业对 AI 集成的精细管控需求：

- 某些企业可能因为数据安全合规要求，只允许使用特定的 AI 模型
- 管理员可以在不影响其他 Copilot 功能的前提下，选择性禁用第三方模型

### 1.7 Proposed API：细粒度工具审批

```typescript
export interface LanguageModelToolConfirmationMessages {
  approveCombination?: string | MarkdownString
}
```

之前审批工具是"全有全无"——批准某个工具就批准了它的所有调用。新 API 支持按**工具 + 参数组合**审批。

**例子**: "Run VS Code Command" 工具可以运行任何命令。用户可以只批准 `editor.action.formatDocument`，但拒绝其他命令。

这是安全模型的重要演进：从粗粒度的"信任/不信任某个工具"到细粒度的"信任某个工具的特定用法"。

### 1.8 Notable Fixes 亮点

- **集成浏览器快捷键冲突修复** (#303908): VS Code 快捷键不再优先于页面快捷键
- **调试时"Add Element to Chat"修复** (#299777): 集成浏览器调试暂停时可以正常添加元素到 chat
- **Merge Editor 优化** (#304404): `removeDiffs` 从 O(K\*N) 优化到 O(N) 单次遍历
- **Timeline 内存泄漏修复** (#304668): 切换面板可见性时的内存泄漏
- **文件描述符泄漏修复** (#303395): 销毁读取流防止 fd 泄漏
- **MCP 环境变量解析** (#303156): agent 插件中 MCP server 定义的环境变量现在可以正确解析

## 二、v1.115：Agent-Native 开发的里程碑

### 2.1 VS Code Agents 伴侣应用（Preview）⭐⭐ 最重要

这是 v1.115 的**标题级功能**。VS Code 推出了一个全新的伴侣应用——**VS Code Agents**。

#### 核心特性

1. **并行化多项目 Agent 会话**
   - 跨多个仓库并行启动 agent 会话
   - 每个会话隔离在独立的 worktree 中（Git worktree）
   - 快速切换上下文，UI 自适应当前选择

2. **监控和审查**
   - 跟踪 session 进展
   - 内联查看 diff
   - 给 agent 留反馈
   - 直接创建 PR，无需离开应用

3. **配置继承**
   - 自定义指令（.instructions.md）
   - Prompt 文件（.prompt.md）
   - 自定义 Agent（.agent.md）
   - MCP servers
   - Hooks 和 Plugins
   - 主题等 VS Code 自定义设置
   - **全部自动继承，无需重新配置**

4. **零安装**
   - 随 VS Code Insiders 一起发布
   - 从开始菜单/应用文件夹启动
   - 或通过命令面板 `Chat: Open Agents Application` 启动

**深度分析**: 这标志着 VS Code 正式分化出两种使用模式：

```
VS Code（编辑器模式）           VS Code Agents（Agent 模式）
┌─────────────────────┐     ┌─────────────────────────────┐
│ 人类主导开发         │     │ Agent 主导开发               │
│ 编辑代码、调试、测试  │     │ 并行多仓库 agent 会话        │
│ AI 作为辅助         │     │ 人类监督和审查                │
│ 单一焦点            │     │ 多任务并行                   │
└─────────────────────┘     └─────────────────────────────┘
```

Agents 应用最大的亮点是**并行化** + **隔离的 worktree**。这意味着：

- 可以同时让 agent 修 bug A（分支 fix-a）、实现 feature B（分支 feat-b）、重构模块 C
- 每个任务互不干扰，各自有独立的工作树
- 开发者的角色从"写代码"变成"分配任务 + Review 结果 + 创建 PR"

这是**Agentic Development**（agent 原生开发）从概念走向产品化的关键一步。

> 目前仅在 VS Code Insiders 中可用，是快速迭代的 Preview 状态。

### 2.2 集成浏览器改进

#### 2.2.1 更好的工具标签

Agent 调用浏览器工具时，之前只显示泛泛的 "Clicked element in browser"。

```
旧: "Clicked element in browser"
新: "Right-clicked header banner in Test Page" + [链接到目标浏览器标签]
```

标签现在描述了**具体操作 + 目标元素 + 页面名称**，并附带直接跳转链接。这大幅提升了 agent 操作的可读性和可审计性。

#### 2.2.2 长时间脚本支持

**设置项**: `workbench.browser.enableChatTools`

`Run Playwright Code` 工具现在支持长时间运行的脚本。超过 5 秒的脚本会返回一个**延迟结果（deferred result）**，agent 可以轮询检查。

```
旧流程: 脚本运行 → 超时 → 失败
新流程: 脚本运行 → 5秒后返回 deferred → agent 轮询 → 获取结果
```

这是异步编程模式在 agent 工具链中的应用，避免了阻塞式等待。

#### 2.2.3 更少的重复标签页

Agent 之前会频繁打开重复的浏览器标签。现在当 agent 试图打开新标签时，如果已有同域名的标签打开，则自动复用，除非 agent 显式传入标志。

**本质**: 资源管理的改进。避免 agent 无限制地打开标签导致内存占用增长。

#### 2.2.4 macOS 捏合缩放

集成浏览器现在支持 macOS 触控板的捏合缩放手势，最大放大 3 倍。与标准浏览器缩放（⌘+/⌘-）不同，这是纯视觉放大，不会触发页面回流（reflow）。

技术细节：标准 zoom 修改的是 CSS 的视口宽度，会导致响应式断点变化和布局重排。捏合缩放仅在渲染层面做缩放变换，性能更好且不改变布局。

### 2.3 终端工具改进

#### 2.3.1 向后台终端发送输入（send_to_terminal）

之前后台终端是**只读**的，只能用 `get_terminal_output` 查看。如果前台终端超时移到后台，agent 就无法再与之交互。

新的 `send_to_terminal` 工具解决了这个问题。

**典型场景**:

```
1. Agent 发起 SSH 连接
2. 等待密码提示 → 前台终端超时
3. 终端移到后台
4. [旧] Agent 无法输入密码 → 卡死
5. [新] Agent 用 send_to_terminal 发送密码 → 继续执行
```

#### 2.3.2 后台终端通知（实验性）

**设置项**: `chat.tools.terminal.backgroundNotifications`

之前 agent 必须手动轮询 `get_terminal_output` 来检查后台命令状态。现在 agent 可以**自动收到通知**：

- 命令执行完毕
- 命令需要用户输入

```
旧模式（轮询）:
Agent → get_terminal_output → 没完成
Agent → get_terminal_output → 没完成
Agent → get_terminal_output → 完成了!

新模式（事件驱动）:
Agent ← 通知: "命令已完成"
Agent: 处理结果
```

**深度分析**: 从轮询到事件驱动是 agent 架构演进的重要一步：

- **减少无效 token 消耗**: 不再浪费 LLM 调用去检查"没完成"
- **降低延迟**: 命令完成后立即响应，不用等下次轮询
- **更接近真实开发者**: 人类开发者也是"等终端有输出了再看"，而不是反复刷新

### 2.4 Notable Fixes 亮点

- **终端 pwsh 重启光标位置错误** (#304257)
- **Caps Lock 在 Claude Code 终端中插入原始转义序列** (#304679): `[57358u` 这种乱码问题
- **灾难性正则回溯修复** (#307447): `_extractImagesFromOutput` 中的正则可能导致 ReDoS
- **评论面板内存泄漏** (#304666): 回收树项时的泄漏
- **minimap 中显示覆盖率指标** (#307250): 新功能，测试覆盖率信息现在在 minimap 中可见

## 三、两个版本的宏观趋势总结

### 3.1 Agent 工具链的完整性

```
v1.114-115 建立的 Agent 工具链:

语义搜索 ────────── #codebase (纯语义)
精确搜索 ────────── grep, text search, symbol
文件操作 ────────── 读/写/创建/编辑
终端操作 ────────── 前台执行 + 后台交互 + 事件通知
浏览器操作 ──────── 页面交互 + Playwright + 缩放
审批控制 ────────── 细粒度工具+参数审批
诊断调试 ────────── /troubleshoot + 历史会话回溯
多项目并行 ──────── VS Code Agents app + worktree 隔离
```

### 3.2 从"AI 辅助编程"到"Agent 原生开发"

| 阶段              | 特征                             | VS Code 版本 |
| ----------------- | -------------------------------- | ------------ |
| 1. 自动补全       | 按 Tab 接受建议                  | Copilot 早期 |
| 2. Chat 对话      | 问答式编程辅助                   | ~v1.80+      |
| 3. Agent 模式     | 自主使用工具、迭代解决问题       | ~v1.100+     |
| 4. **Agent 原生** | **独立 App、并行会话、人类监督** | **v1.115**   |

v1.115 的 Agents 应用标志着正式进入第 4 阶段。

### 3.3 开发者角色演变

```
旧角色: 开发者 = 写代码的人
                  ↓
过渡期: 开发者 = 写代码 + 指挥 AI 的人
                  ↓
新角色: 开发者 = 任务分配者 + Agent 监督者 + 质量审查者
```

VS Code Agents app 的设计（监控会话进展、review diff、给 agent 反馈、创建 PR）完美映射了这个新角色模型。

### 3.4 安全模型持续演进

- 细粒度工具审批 API（v1.114）
- 企业级 Claude agent 禁用策略（v1.114）
- 浏览器标签复用避免资源泄漏（v1.115）
- 后台终端可控交互（v1.115）

每一项都在解决同一个问题：**如何在给 Agent 足够自由的同时保持人类的控制权？**

## 四、开发者行动建议

1. **检查 TypeScript 配置**: TS 6.0 废弃了一些选项，尽快检查 `tsconfig.json` 中是否有待迁移的配置
2. **了解新的语义索引**: 如果之前依赖本地索引，需要等待重新索引完成
3. **尝试 VS Code Insiders + Agents app**: 体验并行 agent 会话的工作流
4. **启用实验性设置**: `chat.tools.terminal.backgroundNotifications` 可以显著改善终端交互体验
5. **利用 /troubleshoot + #session**: 当 Copilot 行为不符合预期时，用这个功能进行事后诊断
6. **关注 Edit Mode 废弃时间线**: v1.125 将完全移除，目前可通过 `chat.editMode.hidden` 临时启用

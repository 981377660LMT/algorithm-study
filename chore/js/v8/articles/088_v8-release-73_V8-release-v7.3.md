# V8 v7.3 发布：零成本异步栈追踪与 Wasm 启动优化

# V8 release v7.3: Zero-cost async stack traces and Wasm startup

V8 v7.3 让开发者在调试异步代码时不再盲人摸象。

### 1. 突破黑盒：零成本异步栈追踪

以前，异步代码（如 `await` 之后的内容）丢回事件循环后，原本的调用栈就会丢失。v7.3 引入了“零成本”异步栈追踪：V8 通过在 `await` 恢复点保留轻量级的索引，在生成错误堆栈时能重新拼接出逻辑上的连续调用链，而不会像之前实验性方案那样增加显著的内存开销。

### 2. 异步性能的最后拼图

默认开启了提升 `await` 响应速度的规范变更。通过缩减 Promise 链路中的 Microtick 数量，`async/await` 终于在大多数基准测试中跑赢了手写的 Promise 链。

### 3. Wasm 分层编译：Liftoff 正式上线

引入了全新的基线编译器 Liftoff。它能在不到一眨眼的时间内完成大型 Wasm 模块的解析和执行准备，虽然生成的代码非极致优化，但它极大地缓解了网页开启 Wasm 游戏时的初始白屏时间。

### 4. 一针见血的见解

v7.3 是开发者体验的胜利。零成本异步栈追踪解决了 JS 异步编程中最痛苦的“调试难”问题，这标志着 V8 正在从单纯追求性能指标，转向提升整个全栈开发者的生产力。

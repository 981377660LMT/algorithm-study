# V8 release v6.2: The end of the road for Full-codegen

V8 v6.2 是一个划时代的版本，它彻底清理了过去十年的技术债。

### 1. 架构里程碑：移除 Full-codegen

在此版本中，V8 删除了超过 30,000 行代码，彻底移除了旧的基线编译器 Full-codegen。这意味着所有的 JavaScript 代码现在都统一走 **Ignition (字节码解释器) -> TurboFan (优化编译器)** 的流水线。这种简化的架构不仅降低了引擎的维护难度，还显著减少了二进制体积和内存占用。

### 2. 并行 Scavenger：垃圾回收的提速

引入了并行化（Parallel）的新生代垃圾回收算法。利用多核 CPU 同步进行对象拷贝（Evacuation），显著缩短了主线程的“停顿”时间。在大型对象池的渲染任务中，这能带来感知明显的流畅度提升。

### 3. Proxy 性能翻倍

由于 TurboFan 的深度介入，`Proxy` 对象的属性存取和函数调用性能提升了 5-6 倍。V8 开始将 Proxy 的拦截逻辑直接内联（Inline）到编译后的代码中，使其逐渐摆脱“性能杀手”的恶名。

### 4. 一针见血的见解

v6.2 象征着 V8 从“双编译器兼容”过渡到了“单一解释器+单一优化编译器”的纯粹状态。这种破旧立新是 V8 能够快速支持后续复杂 ES 规范的前提——不再需要为两个架构重复编写两份实现。

# V8 release v3.15: Parallel Marking experiments

### 1. 探索主线程的解脱

为了应对日益复杂的桌面端 JS 应用，v3.15 开始实验性地将 **Marking**（标记）阶段的一部分工作移出主线程。
虽然此时大部分 GC 逻辑仍然是 STW（Stop-The-World）的，但这种将工作量拆分并并行化的尝试，为后来震惊业界的 **Orinoco** 并发标记机制埋下了伏笔。

### 2. 正则引擎的微调

改进了 Irregexp（V8 的正则引擎）对复杂捕获组的处理，通过更精细的指令调度减少了回溯时的性能剧减。

### 3. 一针见血的见解

高性能引擎的演进往往遵循“先做对，再做快，最后做平滑”的规律。v3.15 正处于从“做快”到“做平滑”的转折点，它开始意识到多核处理器的存在，并试图利用这些闲置算力来掩盖 GC 的天然代价。

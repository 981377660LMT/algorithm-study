# V8 v9.0 发布：正则匹配索引与超快 super 访问

# V8 release v9.0: RegExp Match Indices and Faster Super

V8 9.0 是一个里程碑版本，标志着性能与标准的一次大跨度融合。

### 1. 正则匹配索引 (`/d` 标志)

开发者现在可以获取捕获组在原始字符串中的确切起始和终止坐标。

- **背景**：以前只有匹配内容，没有位置信息。为了获取位置，开发者常需复杂的二次扫描。
- **工程价值**：V8 内部直接在匹配过程中记录位置并放入内存，这对于代码编辑器的高亮显示、语法解析等任务具有极其重要的性能价值。

### 2. JS-to-Wasm Inline Wrappers

这是一个具有深远意义的性能突破。

- **内联包装器**：以前 JS 调用 Wasm 需要经过一个通用的“包装器（Wrapper）”进行参数转换。在 9.0 中，TurboFan 现在可以将这些包装器直接**内联（Inline）**到调用点。
- **消除边界**：这省去了一次函数调用的栈帧开销，使得 JS 与 Wasm 之间的边界变得前所未有的薄弱。

### 3. 一针见血的见解

v9.0 深刻体现了 V8 对“互操作性”的极致追求。无论是 JS 与正则引擎，还是 JS 与 WebAssembly，V8 都在通过消除中间转换层（Middle Layer）来榨取每一毫秒。这种“边界融合”的思路，是现代高性能 Web 应用能够持续复杂化的基石。

# V8 release v4.1: Computing properties and versioning shift

V8 v4.1 标志着项目管理与语言特性的同步升级。

### 1. 动态之力：计算属性名

支持了 `{[key]: value}`。解析器现在必须处理在对象字面量定义时的非字面量键名。
V8 的优化技巧：对于常见的计算属性（如由变量定义的字符串键），引擎会尝试复用其现有的符号缓存（Symbol Cache），以减少对象创建时的哈希抖动。

### 2. 内存安全：WeakSet 的引入

补全了 ES6 的弱引用家族。这要求 GC 能够精准识别这种特殊的引用轨道：即便对象被 `WeakSet` 持有，如果外部没有其他强引用，对象仍然必须被及时回收。

### 3. 一针见血的见解

从 v4.1 开始，V8 的版本号与 Chrome 发布周期完全对齐（Chrome 41 对应 V8 4.1）。这一工程管理上的小动作，实际上极大地降低了 Web 开发者预测引擎能力的门槛，标志着 V8 已成为浏览器发布流水线中高度成熟的标准化模块。

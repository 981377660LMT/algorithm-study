# V8 v5.9 发布：Ignition 与 TurboFan 的全面默认启航

# V8 release v5.9: Ignition and TurboFan by default

这个版本是 V8 历史上最重要的分水岭——新架构全面替代了旧系统。

### 1. 默认启用新流水线

从 v5.9 开始，Ignition（解释器）和 TurboFan（编译器）不再是实验性选项，而是成为了默认的执行引擎。这一转变标志着 V8 从一个依赖“直接基线编译”的系统演进为一个更平衡的“分层解释编译”系统。

### 2. 内存占用的剧降

由于字节码（Bytecode）的大小平均只有机器码的 50% 甚至更少，Chrome 在加载 JS 密集型页面时的内存峰值显著降低。这在当时 1GB-2GB 内存占主流的安卓设备上，极大地降低了渲染进程崩溃（OOM）的概率。

### 3. WebAssembly 的飞跃

对此版本的 Wasm 支持进行了底层机器指令级的打磨。通过在代码生成中更激进地利用寄存器分配，Wasm 的执行稳定性得到了质的提升。

### 4. 一针见血的见解

v5.9 是一个时代的结束，也是一个时代的开始。它宣告了 V8 能够以比以往任何时候都更轻盈的姿态，去承载日益膨胀的现代 JS 特性，为后续 Node.js 在服务器端的极速发展夯实了技术地基。

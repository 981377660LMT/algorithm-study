# V8 v7.2 发布：嵌入式内置函数与展开运算符优化

# V8 release v7.2: Embedded builtins and spread operator speedups

V8 v7.2 继续在减少内存占用和提升常用语法速度上发力。

### 1. 技术底盘：嵌入式内置函数 (Embedded Builtins)

V8 拥有数以千计的底层工具函数（如数组操作、Promise 管理）。以往每个 V8 实例（Isolate）都会在内存里拷贝一份这些函数的编译结果。v7.2 让这些代码直接驻留在可执行文件中，被所有上下文共享。
结果是：每个 Chrome 标签页的内存占用平均节省了约 200KB-1MB，这在大规模多标签页场景下极具价值。

### 2. 语法加速：展开运算符 `[...]`

展开运算符的性能在之前版本一直被抱怨。v7.2 通过检测输入数组是否为“快速数组”（具备连续内存布局且无复杂 getter），直接绕过了昂贵的 JS 迭代器协议，改用高效的底层内存拷贝。这不仅让 `[...array]` 快了数倍，也让基于此的函数调用（Spread Call）获益。

### 3. 解析加速

通过重构解析器（Parser）的缓存策略，减少了对重复加载脚本的冗余扫描。对于 Facebook 这种巨型网站，页面的冷启动解析速度提升了约 30%。

### 4. 一针见血的见解

v7.2 是“工程化”的胜利。嵌入式内置函数彻底解决了 V8 在多实例环境下的“血统重复”问题，也为 JS 应用的轻量化提供了标准范式。

# 原生特快：内建函数的 C++ 化

# V8 release v3.7: C++ builtins and native integration

### 1. 消除脚本层冗余

许多早期的 JS 内建函数（如数组的部分方法）是用 JS 实现的。这虽然方便跨平台，但由于 JS 自身的动态性，调用它们会产生额外的开销。
v3.7 将更多的核心逻辑迁移到 C++ 原生实现，并通过 V8 的 **Fast API Call** 机制，实现了 JS 和 C++ 之间的透明调用，极大地提升了标准库的响应速度。

### 2. 构建系统的变革

V8 开始为了更好的跨平台编译体验改进构建脚本。

### 3. 一针见血的见解

底层的统一才是真正的强大。v3.7 展示了 V8 团队的一种思维：如果这种逻辑是所有应用都避不开的“公分母”，那么就应该把它作为引擎的原语来实现。这种“下沉”式优化为后来 JS 引擎标准库的飞跃奠定了架构基础。

# V8 v4.6 发布：并发清除与扩展运算符的预演

# V8 release v4.6: Concurrent sweeping and spread in arrays

V8 v4.6 在并发回收（Orinoco）的深度和语法便捷性上同步前进。

### 1. 垃圾回收：并发清除 (Concurrent Sweeping)

在此版本中，V8 将“清除”（Sweeping）步骤从主线程中彻底剥离，交由专门的后台线程处理。当标记完成后，主线程几乎可以立即恢复执行 JS，而后端的清除工作会在不产生主线程停顿的情况下静默完成。这是消灭长耗时 GC 任务的关键。

### 2. 语法力量：数组扩展运算符

正式引入了数组字面量中的 `...` 扩展语法。为了保证性能，V8 对小型数组的扩展做了内联优化，避免了不必要的昂贵迭代调用。

### 3. 一针见血的见解

v4.6 展示了 V8 的多核野心。并发清除的引入是引擎向“真并发回收”迈进的重要一步。它告诉开发者，JS 性能的提升不一定来自更快的执行速度，往往也来自更少的主线程打断。

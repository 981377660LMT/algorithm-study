# V8 release v4.7: Array.prototype.includes and memory pressure optimizations

V8 v4.7 在引入便利特性的同时，开始着重于在极端负载下的垃圾回收调度。

### 1. 技术前瞻：`Array.prototype.includes`

率先引入了当时还处于 Stage 3 的 `includes` 方法。这比传统的 `indexOf !== -1` 不仅语义更清晰，更重要的是，它能正确处理 `NaN` 的查找（NaN 不等于自身），是第一个原生支持该语义的搜索方法。

### 2. 回应内存压力

优化了在重负载页面的垃圾回收触发频率。通过更细致的分配统计，V8 开始有意识地推迟某些不必要的 GC，以确保在动画和密集交互期间减少主线程的突发停顿。

### 3. 一针见血的见解

v4.7 的核心价值在于其“用户侧感知”。无论是更好用的数组搜索，还是更平滑的 GC 触发逻辑，都体现了 V8 对真实世界中复杂网页应用（特别是具有大量定时任务的页面）在性能平衡点上的精准拿捏。

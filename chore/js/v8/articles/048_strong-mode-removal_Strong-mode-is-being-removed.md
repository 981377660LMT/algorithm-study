# 弃用“强模式”：V8 对高性能方言尝试的失败教训

# Strong mode is being removed: A lesson in engine design

这是一篇极具启发性的技术反思，展示了 V8 如何在尝试改变 JS 语言特性以适应引擎优化。

### 1. 什么是强模式 (Strong Mode)？

曾是 V8 试图在 `use strict` 基础上推出的更严格子集：要求变量必须初始化、禁止越界数组访问、禁止 `delete` 等。目的是通过消除动态特性的“不可预测性”，让当时的优化器 Crankshaft 获得更强的静态假定。

### 2. 失败原因：生态阻力和 TurboFan 的崛起

- **收益不明显**：实测发现，强制静态化并没有带来预想中的大幅提速。
- **迁移代价大**：现有 JS 代码库迁移到强模式非常痛苦，开发者接受度极低。
- **架构替代**：新的 TurboFan 编译器证明了它完全有能力处理高度动态的代码，不再需要通过限制开发者来换取性能。

### 3. 一针见血的见解

Strong Mode 的失败确立了 V8 的核心价值观：**不应该让语言适应引擎，而应该让引擎适应语言**。这一决策最终导向了后来全能型的 TurboFan 架构，而不是分裂出一种“高性能 JS 方言”。

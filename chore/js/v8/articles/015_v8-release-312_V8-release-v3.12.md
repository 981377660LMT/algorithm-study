# V8 release v3.12: Pre-parser improvements

### 1. 应对巨型库文件的冷启动

随着 jQuery、Underscore 等类库的体积不断膨胀，冷启动时的解析压力（Parsing Time）日益突显。
v3.12 改进了 **Pre-parser**。它可以在初始阶段跳过那些在启动时并不执行的函数体，只记录函数名和参数。等到函数被真正调用时再进行全量解析（Full Parse）。优化点在于降低了预解析时的 AST 生成开销，让首页执行能够更早开始。

### 2. 垃圾回收的页级回收

增强了 Old Space（老生代）的内存清理效率，允许 GC 能够以更细的粒度（Page 级别）将连续的空闲内存块归还给操作系统。

### 3. 一针见血的见解

“最好的代码是没有运行过的代码”。V8 通过不断精进的惰性解析（Lazy Parsing）平衡了“加载”与“执行”之间的矛盾。只有在真正需要时才投入算力，是 V8 保持轻量化、响应迅速的关键策略。

# V8 release v3.4: Early performance benchmarks and dominance

在 Crankshaft 刚刚问世的这个时期，V8 在各种基准测试（如 SunSpider, V8 Benchmark Suite）中几乎是一骑绝尘。

### 1. 消除解释器的代价

在 2008 年到 2011 年这段时间，V8 有一个非常激进的特点：**它没有解释器**。
即使是第一行执行的代码，V8 也会通过 **Full-codegen** 迅速将其转化为机器码。这种策略在当时的硬件环境下虽然激进且消耗内存，但对于提升交互响应速度起到了决定性作用。

### 2. 精炼的指令生成

这个版本的优化点在于减少生成的机器码体积。由于没有解释器，代码膨胀是一个严峻问题。V8 通过重构常用操作的代码生成模板，实现了性能提升与内存占用下降的双赢。

### 3. 一针见血的见解

早期的 V8 是一种“暴力美学”的体现。它通过直接生成极致精简的机器码，强行拉升了 Web 应用的性能下限。这种先发制人的策略，不仅赢得了基准测试，更赢得了开发者对 Web 平台的信心。
